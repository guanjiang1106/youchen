# 界面重构功能增强说明

## 功能概述

在原有的"界面重构"功能基础上，新增了两个重要功能：

1. **新增按钮功能**：当启用"保留原功能"开关时，可以选择性地添加一个自定义按钮及其功能
2. **单文件重新生成**：当某个文件生成有错误时，可以针对该文件单独重新生成，无需重新生成所有文件

## 修改内容

### 1. 前端修改 (refactorDialog.vue)

#### 新增表单项：

1. **新增按钮开关**
   - 位置：在"保留原功能"开关下方
   - 仅在"保留原功能"启用时可见
   - 用于控制是否启用新增按钮功能

2. **按钮名称输入框**
   - 仅在"新增按钮"启用时可见
   - 用于输入按钮的显示名称
   - 示例：批量导出、数据统计、快速审核

3. **按钮功能描述输入框**
   - 仅在"新增按钮"启用时可见
   - 多行文本输入框
   - 用于详细描述按钮的功能需求
   - 系统会根据描述分步生成：Controller接口 → Service逻辑 → 前端按钮和对话框

#### 新增重新生成功能：

每个代码标签页（index.vue、controller.py、service.py、api.js）都新增了：

1. **重新生成按钮**
   - 位置：在每个标签页的操作按钮区域
   - 点击后弹出对话框，让用户输入问题描述
   - 支持描述语法错误、功能缺失、逻辑调整等问题

2. **错误信息显示区域**
   - 位置：在代码显示区域下方
   - 当重新生成失败时，显示详细的错误信息
   - 红色警告框，便于用户查看问题

3. **加载状态**
   - 每个文件独立的加载状态
   - 重新生成时显示加载动画
   - 避免重复点击

## 使用流程

### 1. 完整重构流程

#### 启用新增按钮功能

1. 打开"界面重构"对话框
2. 确保"保留原功能"开关已启用
3. 启用"新增按钮"开关
4. 填写按钮名称（例如：批量导出）
5. 填写按钮功能描述

#### 功能描述示例

```
1. 点击按钮后弹出对话框
2. 对话框中显示选中数据的统计信息
3. 支持导出统计结果为Excel
4. 包含数据验证和错误提示
```

#### 生成结果

系统会分步生成以下文件：

1. **Controller 接口** - 只返回新增的方法（不包含完整类）
2. **Service 逻辑** - 只返回新增的方法（不包含完整类）
3. **前端 Vue 文件** - 完整代码（包含所有原有功能和新增按钮）
4. **前端 API 文件** - 只返回新增的 API 方法

### 2. 单文件重新生成流程

当某个文件生成有错误时：

#### 步骤 1：发现问题

在重构结果对话框中，查看生成的代码，发现某个文件有问题，例如：
- Controller 代码有语法错误
- Service 逻辑不完整
- 前端页面缺少某个功能

#### 步骤 2：点击重新生成

1. 切换到有问题的标签页（如 controller.py）
2. 点击"重新生成"按钮（黄色警告按钮）
3. 在弹出的对话框中，详细描述问题

#### 步骤 3：描述问题

问题描述示例：

```
当前代码存在以下问题：
1. 缺少 try-except 异常处理
2. 方法参数顺序不对，非默认参数在默认参数后面
3. 缺少数据验证逻辑
4. 返回值类型不正确

请修复这些问题，并确保代码可以直接运行。
```

#### 步骤 4：查看结果

- 系统会根据问题描述重新生成该文件的代码
- 只更新当前文件，不影响其他文件
- 如果仍有错误，会在代码下方显示错误信息
- 可以多次重新生成，直到满意为止

#### 步骤 5：应用代码

确认所有文件都正确后，点击"应用重构"按钮

### 3. 重新生成的优势

1. **精准修复**：只针对有问题的文件重新生成，不影响其他正确的文件
2. **快速迭代**：无需重新生成所有文件，节省时间
3. **问题追踪**：可以针对具体问题进行描述，AI 更容易理解和修复
4. **多次尝试**：可以多次重新生成同一个文件，直到满意
5. **错误提示**：失败时显示详细错误信息，便于调整描述

## 技术要点

### AI 生成策略

1. **前端代码**：返回完整的 Vue 文件，保留所有原有功能
2. **后端代码**：只返回新增的方法，不返回完整类定义
3. **权限标识**：自动生成格式为 `{module_name}:{business_name}:{button_name}`

### 代码格式要求

1. Python 代码使用 ```python 代码块包裹
2. 正确的缩进（4空格为一级）
3. 不包含 import 语句和类定义
4. 只包含新增的方法

### 示例输出格式

```json
{
  "index": "完整的index.vue代码（包含所有原有功能和新增按钮）",
  "controller": "```python\n    @classmethod\n    async def new_method(cls, ...):\n        pass\n```",
  "service": "```python\n    @classmethod\n    async def new_method_services(cls, ...):\n        pass\n```",
  "api": "export function newApi(data) {\n  return request({\n    url: '/path',\n    method: 'post',\n    data: data\n  })\n}"
}
```

## 注意事项

1. **保留原功能**：必须启用"保留原功能"才能使用新增按钮功能
2. **功能描述**：描述越详细，生成的代码越准确
3. **问题描述**：重新生成时，问题描述要具体明确，包含：
   - 具体的错误信息
   - 缺少的功能
   - 需要调整的逻辑
   - 期望的结果
4. **分步实现**：复杂功能建议分多次提交，每次实现一个小功能
5. **代码检查**：生成后建议先检查代码，确认无误后再应用
6. **多次尝试**：如果一次生成不满意，可以多次重新生成
7. **保存进度**：重新生成只更新指定文件，其他文件保持不变

### 新增按钮功能

3. **灵活可控**：可以在应用前编辑生成的代码

2. **节省时间**：无需重新生成所有文件
3. **问题追踪**：可以针对具体问题进行描述
4. **多次迭代**：支持多次重新生成，直到满意

## 使用场景

### 场景 1：生成代码有语法错误

**问题**：Controller 代码生成后有缩进错误

**解决方案**：
1. 切换到 controller.py 标签页
2. 点击"重新生成"按钮
3. 描述问题："代码有缩进错误，try-except 块的缩进不正确，请修复"
4. 查看重新生成的代码
5. 如果还有问题，继续重新生成

### 场景 2：功能不完整

**问题**：Service 代码缺少数据验证逻辑

**解决方案**：
1. 切换到 service.py 标签页
2. 点击"重新生成"按钮
3. 描述问题："当前代码缺少数据验证，请添加：1. 检查参数是否为空 2. 检查数据是否存在 3. 检查权限"
4. 查看重新生成的代码

### 场景 3：需要调整逻辑

**问题**：前端页面的按钮位置不对

**解决方案**：
1. 切换到 index.vue 标签页
2. 点击"重新生成"按钮
3. 描述问题："新增的按钮应该放在表格上方的操作栏，而不是表格内部"
4. 查看重新生成的代码

### 场景 4：多个文件都有问题

**解决方案**：
1. 先修复 Service 层（业务逻辑）
2. 再修复 Controller 层（接口定义）
3. 最后修复前端页面（界面展示）
4. 每个文件单独重新生成，互不影响
