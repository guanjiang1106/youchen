# "保留原功能"模式说明

## 功能概述

界面重构功能提供两种模式：

### 1. 保留原功能模式（推荐）✅

**勾选"保留原功能"时**：
- ✅ 保留所有增删改查功能
- ✅ 保留所有表格、表单、对话框
- ✅ 保留所有字段和验证规则
- ✅ 保留所有字典数据定义
- ✅ 保留所有方法和事件处理
- ✅ 保留所有权限控制
- ✅ 在原有基础上增强和优化

**适用场景**：
- 优化现有界面的样式和布局
- 添加新功能（如新增按钮）
- 改进用户体验
- 增强交互效果
- 添加新的数据展示方式

**需求描述示例**：
```
1. 优化表格样式，使用斑马纹
2. 添加搜索框的快捷键支持
3. 优化表单布局，使用两列布局
4. 添加数据导出功能
5. 增加批量操作按钮
```

### 2. 完全重构模式

**不勾选"保留原功能"时**：
- ❌ 不保留任何原有功能
- ✅ 根据需求完全重新设计
- ✅ 可以实现全新的界面
- ✅ 适合完全不同的需求

**适用场景**：
- 完全重新设计界面
- 实现全新的功能
- 原有功能不再需要
- 需要全新的数据结构

**需求描述示例**：
```
创建一个数据看板页面，包含：
1. 顶部显示4个统计卡片
2. 中间显示折线图和柱状图
3. 底部显示最近的数据列表
4. 不需要增删改查功能
```

## 保留原功能的技术实现

### 1. 读取原有代码

系统会自动读取原有的 Vue 文件：

```python
# 构建文件路径
vue_file = os.path.join(frontend_path, 'src', 'views', module_name, business_name, 'index.vue')

# 读取原有代码
if os.path.exists(vue_file):
    async with aiofiles.open(vue_file, 'r', encoding='utf-8') as f:
        original_code = await f.read()
```

### 2. 传递给 AI

将原有代码包含在 AI 提示词中：

```python
system_prompt += f"""
【原有代码】
```vue
{original_code}
```

【重要要求 - 保留原功能模式】
1. 必须保留原有代码中的所有增删改查功能
2. 必须保留原有的表格、表单、对话框等核心组件
3. 必须保留原有的所有字段和验证规则
...
"""
```

### 3. AI 生成增强代码

AI 会：
- 分析原有代码的结构
- 保留所有核心功能
- 根据需求添加新功能
- 优化样式和交互
- 保持代码风格一致

## 字典数据保留

### 问题

原有代码中的字典数据定义和加载逻辑容易被 AI 忽略，导致前端报错。

### 解决方案

在 AI 提示词中特别强调：

```
【特别注意 - 字典数据】
1. 必须保留原有代码中所有的字典数据定义，例如：
   - sys_user_sex: []
   - sys_normal_disable: []
   - 其他字典数据
2. 必须保留原有的 getDicts() 调用，例如：
   - this.getDicts("sys_user_sex", "sys_normal_disable").then(...)
3. 不要删除或修改任何字典相关的代码
```

### 验证

生成代码后，检查：
- `data()` 中是否包含所有字典数据定义
- `created()` 或 `mounted()` 中是否调用了 `getDicts()`
- 模板中使用的字典数据是否都已定义

## 新增按钮功能

### 启用条件

只有在"保留原功能"模式下，才能勾选"新增按钮"。

### 生成内容

勾选"新增按钮"后，会生成 4 个文件的代码：

1. **index.vue**：完整的前端 Vue 文件（包含新按钮）
2. **controller.py**：新增的 Controller 方法
3. **service.py**：新增的 Service 方法
4. **api.js**：新增的前端 API 方法

### 代码追加

- Vue 文件：覆盖写入
- Controller 文件：追加到文件末尾
- Service 文件：追加到文件末尾（自动修复缩进）
- API 文件：追加到文件末尾

详见：`NEW_BUTTON_FEATURE.md`

## 使用建议

### 1. 优先使用保留模式

除非需要完全重新设计，否则建议始终勾选"保留原功能"。

### 2. 明确需求描述

在需求描述中明确说明：
- 要优化什么
- 要添加什么
- 要改进什么
- 不要说"删除XXX功能"

### 3. 预览代码

应用前仔细预览生成的代码，确认：
- 原有功能是否保留
- 字典数据是否完整
- 新功能是否正确实现

### 4. 使用版本控制

```bash
# 应用前备份
git add .
git commit -m "应用重构前的备份"

# 如果有问题，回滚
git checkout .
```

## 常见问题

### Q1：保留模式下，原有功能被删除了？

**原因**：AI 理解错误或需求描述不当

**解决方案**：
1. 在需求中明确说明"保留所有原有功能"
2. 不要使用"删除"、"移除"等词语
3. 重新生成代码

### Q2：字典数据丢失？

**原因**：AI 没有保留字典相关代码

**解决方案**：
1. 查看原有代码，找到字典定义
2. 手动添加到新代码中
3. 详见 `DICT_DATA_ISSUE.md`

### Q3：新增的功能与原有功能冲突？

**原因**：代码结构不兼容

**解决方案**：
1. 简化需求，分步实现
2. 手动调整代码
3. 使用版本控制回滚

## 最佳实践

### 1. 渐进式增强

不要一次性添加太多功能，建议：
- 第一次：优化样式和布局
- 第二次：添加一个新按钮
- 第三次：添加另一个新功能

### 2. 详细的需求描述

好的需求描述：
```
1. 在表格上方添加一个"批量导出"按钮
2. 点击后弹出对话框，让用户选择导出格式（Excel/PDF）
3. 导出当前筛选条件下的所有数据
4. 保留原有的所有功能，包括增删改查、单条导出等
```

不好的需求描述：
```
添加导出功能
```

### 3. 测试流程

1. 应用代码
2. 重启服务
3. 测试原有功能（增删改查）
4. 测试新增功能
5. 检查字典数据是否正常
6. 检查权限控制是否正常
